# SEO On-Page Blueprint (English Copy, Indonesia Market)

## Purpose
This blueprint keeps the site fully English for premium positioning while preserving strong Indonesian search intent.

## Positioning Rules
- Primary language stays English on all pages.
- Market signal stays Indonesia through entities, locations, and problem context.
- Use Indonesian intent terms selectively in headings, FAQs, and internal links when natural.

## URL and Metadata Standard
- Use one canonical URL per page.
- Use one language signal for now: `en-ID`.
- Keep `x-default` pointing to the same canonical URL.

## Title Formula
Use one of these formulas.

1. Product/Service page
`Primary Solution + Indonesia | Benefit + Brand`

Example:
`Vacuum Lifter Indonesia | Safer Ergonomic Material Handling | PT Dimensi Quantum Wahyudi`

2. Article page
`Problem + Industrial Context Indonesia | Practical Guide`

Example:
`How to Reduce Manual Lifting Risk in Indonesian Manufacturing | Practical Guide`

## Meta Description Formula
- 140 to 160 characters.
- English copy with Indonesia market context.
- Include one clear business benefit and one technical intent.

Template:
`Premium [solution] for manufacturers in Indonesia. Improve [safety/productivity metric] with engineering-backed [system type] and practical implementation guidance.`

## H1 and Intro Pattern
- H1 in English, premium and specific.
- Opening paragraph includes:
  - Indonesia context
  - industrial use-case
  - expected measurable outcome

Example opening:
`Designed for manufacturers in Indonesia, this solution improves operator safety and material flow reliability in high-frequency production environments.`

## FAQ Block Pattern (SEO + Conversion)
Include 3 to 5 FAQs per important page.

Suggested FAQ intent mix:
1. Cost/ROI question
2. Safety/compliance question
3. Technical fit question
4. Implementation timeline question
5. Service/support in Indonesia question

## Internal Linking Rules
- Every article links to at least 2 relevant product pages.
- Every product page links to at least 1 project case and 1 supporting article.
- Use descriptive anchors with commercial intent.

Good anchor examples:
- `vacuum lifter solution for carton handling`
- `ergonomic lift assist system for Indonesian factories`
- `project example in FMCG production line`

## Content Freshness Rules
- Add `Last updated` date for article updates.
- For each updated article, revise:
  - one statistic or benchmark
  - one internal link
  - one implementation recommendation

## Entity and Local Trust Signals
Keep these visible across key pages.

- Company legal name
- Indonesia service coverage
- Contact details and WhatsApp CTA
- Evidence assets (project gallery, technical examples, certifications)

## Editorial Checklist (Before Publish)
1. Title uses formula and includes clear solution intent.
2. Description is 140 to 160 characters and market-specific.
3. H1 and intro mention Indonesia context naturally.
4. FAQ includes technical and commercial intent.
5. At least 2 internal links and no broken links.
6. Images have descriptive alt text in English.
7. Schema is valid and matches visible page content.
